Generated by SURFACE-REMEDIATION-001 from FORENSIC-AUDIT-SURFACES-002 findings.

## Cross-Service Variables (appear in multiple services)

These variables require alignment when changed — updating one service without the other causes drift.

| Variable | Services | Sensitivity | Notes |
|----------|----------|-------------|-------|
| `APP_ENV` | backend, agent-api | config | Controls environment mode (development/staging/production) |
| `DATABASE_URL` | backend, agent-api | config | Connection string; different DBs per service (zakops vs zakops_agent) |
| `DASHBOARD_SERVICE_TOKEN` | dashboard, agent-api | **secret** | Service-to-service auth; must match between caller and callee |
| `DEAL_API_URL` | backend, agent-api | config | Backend API URL for internal calls |
| `DEBUG` | backend, agent-api | runtime | Debug mode flag |
| `LOG_FORMAT` | backend, agent-api | runtime | Logging format (json/console) |
| `LOG_LEVEL` | backend, agent-api | runtime | Logging verbosity |
| `OPENAI_API_KEY` | backend, agent-api | **secret** | LLM API key |
| `POSTGRES_DB` | backend, agent-api | config | Database name (different per service) |
| `POSTGRES_PASSWORD` | backend, agent-api | **secret** | Database password |
| `POSTGRES_PORT` | backend, agent-api | config | Usually 5432 |
| `POSTGRES_USER` | backend, agent-api | config | Database user (different per service) |
| `RAG_REST_URL` | backend, agent-api | config | RAG service endpoint |
| `ZAKOPS_API_KEY` | dashboard, backend, agent-api | **secret** | API authentication key (3-way shared) |

## Naming Conflicts (same concept, different names)

### Backend URL Discovery
The dashboard uses 5 different variables to discover the backend:

| Variable | Used In | Purpose |
|----------|---------|---------|
| `API_URL` | middleware.ts, next.config.ts | Primary backend URL (canonical) |
| `BACKEND_URL` | middleware.ts | Fallback for API_URL |
| `NEXT_PUBLIC_API_URL` | docker-compose build args | Client-side visible backend URL |
| `DEAL_API_URL` | agent-api config | Agent-api → backend calls |
| `ZAKOPS_BACKEND_URL` | backend config | Backend self-reference |

**Resolution:** `API_URL` is canonical for dashboard. Variable renaming deferred to Surface 11.

### Agent API URL Discovery
| Variable | Used In | Purpose |
|----------|---------|---------|
| `AGENT_LOCAL_URL` | dashboard route handlers | Agent API base URL |
| `AGENT_API_URL` | some dashboard routes | Alias for AGENT_LOCAL_URL |
| `NEXT_PUBLIC_AGENT_API_URL` | docker-compose build args | Client-side visible agent URL |

### RAG URL Discovery
| Variable | Used In | Purpose |
|----------|---------|---------|
| `RAG_REST_URL` | agent-api, backend | RAG service URL |
| `RAG_API_URL` | backend | Alias for RAG_REST_URL |
| `ZAKOPS_RAG_API_URL` | backend | Another alias |
| `RAG_ENDPOINT` | backend | Yet another alias |

### vLLM URL
| Variable | Used In | Purpose |

| `VLLM_BASE_URL` | agent-api | vLLM inference endpoint |
