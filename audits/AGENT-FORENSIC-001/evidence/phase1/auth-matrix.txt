AGENT-FORENSIC-001 Phase 1 — Step 1.2: Auth Enforcement Matrix
================================================================
Date: 2026-02-02

AUTH MECHANISMS:
1. Bearer JWT (chatbot/auth endpoints) — HS256, JWT_SECRET_KEY
2. X-Service-Token (chatbot endpoints) — static token for Dashboard
3. Agent JWT (agent approve/reject) — iss/aud/role claims, OPTIONAL
4. None (agent invoke/approvals/threads) — completely open

KEY FINDING: AGENT_JWT_ENFORCE defaults to "false"
  → All agent endpoints accept unauthenticated requests

CRITICAL FINDINGS:
[FAIL] Agent /invoke endpoint requires NO authentication — anyone can invoke the LLM
[FAIL] Agent /approvals listing requires NO authentication — data leakage
[FAIL] Agent /threads/{id}/state requires NO authentication — thread state visible to anyone
[FAIL] Agent approve/reject endpoints accept actor_id from request body when JWT not enforced
       → Actor identity can be spoofed (line 237: actor_id = user.subject if user else action_request.actor_id)
[WARN] AGENT_JWT_ENFORCE=false by default — all agent JWT checks are bypassed
[INFO] Chatbot endpoints properly require Bearer token or X-Service-Token
[INFO] Auth endpoints (session/sessions) properly require Bearer token
