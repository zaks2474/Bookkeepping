4. FIELD COMPLETENESS
---------------------
- actor_id: REQUIRED, NOT NULL - Always populated (forensic-auditor in samples)
- event_type: REQUIRED, NOT NULL with CHECK constraint - All valid event types
- thread_id: Optional - Populated when relevant
- approval_id: Optional - Populated for approval-related events
- tool_execution_id: Optional - Populated for tool execution events
- payload: REQUIRED, NOT NULL with JSONB default - Contains context data

5. EVENT TYPE CONSTRAINT
------------------------
CHECK constraint enforces only valid event types:
- approval_created
- approval_claimed
- approval_approved
- approval_rejected
- approval_expired
- tool_execution_started
- tool_execution_completed
- tool_execution_failed
- stale_claim_reclaimed

NOTE: Schema does not include "target" column - no target field defined.
This is by design; the target is tracked via thread_id, approval_id, or tool_execution_id.
