## Critical Findings

### 1. Backend ResponseValidationError (CRITICAL)

**Symptom:** GET /api/deals and GET /api/quarantine return HTTP 500

**Root Cause:** Response model requires `email_thread_ids` field but database model doesn't include it.

**Error:**
```
fastapi.exceptions.ResponseValidationError: 2 validation errors:
  {'type': 'missing', 'loc': ('response', 0, 'email_thread_ids'), 'msg': 'Field required'}
```

**Impact:** 
- Cannot list deals via API
- Dashboard deal list likely broken
- Deals CREATE works (data persists to DB)

**Fix Required:** Update Pydantic model to match database schema or add `email_thread_ids` column.

### 2. RAG top_k Parameter Ignored (Medium)

**Symptom:** Request with `top_k=3` returns 5 results

**Impact:** Client cannot control result count
