## Architecture

The email ingestion pipeline works as follows:
1. Timer triggers service hourly
2. Service runs email_triage_agent.run_once module
3. Module uses Gmail MCP to fetch new emails
4. Emails classified and sent to quarantine via POST /api/quarantine
5. User approves in UI -> R2-3a.1 creates deal atomically
