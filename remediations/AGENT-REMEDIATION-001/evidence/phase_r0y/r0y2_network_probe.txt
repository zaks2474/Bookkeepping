==============================================================================
SECTION 3: DATA FLOW VERIFICATION
==============================================================================

Dashboard → Agent API Communication:
1. Browser makes request to /api/chat/complete (POST)
2. Dashboard server-side code reads DASHBOARD_SERVICE_TOKEN from process.env
3. Dashboard creates new fetch() to Agent API at http://localhost:8095
4. Token is ADDED to X-Service-Token header for the OUTBOUND request
5. Agent API responds to Dashboard
6. Dashboard returns response to browser WITHOUT including the token

Critical Boundary: The X-Service-Token header is ONLY used for server-to-server
communication (Dashboard → Agent API). It is NEVER included in any response sent
back to the browser.

PASS - No token leakage detected:

Evidence:
1. Response headers from /api/* routes never contain X-Service-Token
2. No Authorization or secret headers exposed in any HTTP response
3. Token only used for server-side outbound authentication (to Agent API)
4. Token completely hidden from browser in all test scenarios
5. All response bodies contain only application data, no secrets

Conclusion: The DASHBOARD_SERVICE_TOKEN is properly encapsulated on the server
side and never transmitted back to the client.
